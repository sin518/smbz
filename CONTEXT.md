# CONTEXT.md

## 项目领域

本项目是一个综合性的中国传统占术（命理）在线服务平台，涵盖多种占术系统。

## 核心占术系统

### 八字（四柱命理）
基于出生年月日时的天干地支排盘，分析命局、大运、流年。包含十神、神煞、格局、喜用神等概念。

### 紫微斗数
基于出生时间和农历日期的斗数排盘系统，涉及十二宫位、主星、辅星、四化（化禄、化权、化科、化忌）等。

### 奇门遁甲
时空占卜系统，基于节气、时辰排出九宫格局，包含天盘、地盘、人盘、神盘等多层信息。

### 六爻
摇卦占卜系统，通过卦象（六个爻位）、世应、六亲、六神等要素进行分析。

### 大六壬
基于天干地支的时空占卜体系，涉及四课、三传、天将等复杂推演。

## 关键业务规则

1. **确定性计算 vs AI 解读分离**：排盘计算（天干地支、星曜位置、卦象生成）必须是确定性的、可复现的；AI 解读是独立的增值服务层。

2. **前端计算优先**：占术排盘逻辑在前端（`packages/core`）完成，避免服务端计算负载，提升用户体验。

3. **用户数据隔离**：每个用户只能访问自己创建的盘局、解读记录，后端 API 必须严格按 user_id 过滤。

4. **会员与积分体系**：
   - 免费用户：有限次数的 AI 解读
   - 会员用户：更多或无限次数的 AI 解读
   - 积分扣减：AI 解读消耗积分，必须在调用 AI 前扣减，失败时退款

5. **多端一致性**：Web 端和移动端（响应式）必须展示一致的排盘结果和解读内容。

## 技术架构关键点

- **Frontend**: Next.js App Router、React Server Components 为主、客户端组件按需使用
- **Backend**: FastAPI、asyncpg（直接 SQL）、可选 Redis 缓存
- **AI 层**: 通过 `divination-pipeline.ts` 统一管理 AI 解读流程（鉴权→积分→AI 调用→持久化）
- **数据库**: Supabase PostgreSQL，使用参数化 SQL，禁止拼接用户输入

## 常见术语

- **排盘**：根据用户输入（出生时间、起卦信息等）生成占术图表的过程
- **盘局**：一次排盘的结果数据
- **解读/解盘**：对排盘结果进行文字化分析和解释
- **AI 解读**：使用大语言模型对盘局进行智能分析
- **天干地支**：中国传统历法的时间表示系统（甲乙丙丁...子丑寅卯...）
- **节气**：二十四节气，用于确定某些占术系统的时间边界
- **农历/阴历**：中国传统历法，与公历（阳历）需要转换
