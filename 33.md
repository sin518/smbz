## 1. “主用神：火、木”顺序建议改

你写：

> 主用神：火、木

但从结构看：

- 子月寒湿，确实需要火调候；
- 但水旺克火，直接补火容易被水压；
- 木能走“水 → 木 → 火”的通关路线。

更稳写法：

```
主用神：木
调候用神：火
条件用神：土
喜神：木、火
忌神：水、金
```

或者：

```
用神体系：以木为通关核心，以火为调候核心。
```

## 2. “年柱己伤官、时柱戊食神透出，且月令/天干见正官结构”这句有错

正官在**月令地支子中癸水**，不是天干透出。

你写“月令/天干见正官结构”容易误导。

建议改成：

```
年干己伤官、时干戊食神透出，月令子水正官当令，存在食伤与官星相见的结构风险。
```

不要写“天干见正官”。

## 3. “子子夹日支寅”这个说法不够规范

月支子、时支子确实在日支两侧，但严格说不是传统固定术语。

可以保留意思，但建议换成更稳的表达：

```
月支、时支两见子水，分居日支两侧，官星压力靠近日主与日支。
```

这样更专业，避免 AI 把它当成固定格局。

## 4. “官杀/克泄压力”仍有残留

你前面已改成“正官压力偏旺”，但后面还有：

> 帮扶与官杀/克泄压力交叉判断

建议改成：

```
帮扶、正官压力与食伤泄身交叉判断
```

避免又引回“官杀混杂”。

## 5. 大运年份最好统一为“起始年份”而不是“起止年份”

你写：

> 乙亥 1998–2008
>  甲戌 2008–2018

严格大运交接通常不是整年 1 月 1 日，而是具体起运月日。你这里用于 AI 模板可以简化，但建议注明：

```
年份为简化区间，实际交运需以具体起运时间为准。
```

## 7. 辛未大运不能只写“风险：金”

未中有乙木、丁火、己土，不是纯土，也不是单纯金土。

建议改成：

```
辛未运：
有利：未中藏乙丁，可补木火；未土适量可制水。
风险：辛金透出可生水，土过旺亦会泄火。
```
